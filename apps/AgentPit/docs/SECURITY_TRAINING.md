# AgentPit 团队安全培训文档

## 1. 培训概述

### 1.1 培训目标
- 提升团队成员的安全意识
- 掌握安全编码规范和最佳实践
- 熟悉安全工具的使用方法
- 了解常见安全漏洞类型及防范措施

### 1.2 培训对象
- 开发团队所有成员
- 代码审查人员
- 项目管理人员

### 1.3 培训时长
- 理论培训：2小时
- 实践操作：1小时

## 2. 安全改造项目回顾

### 2.1 项目背景
- AgentPit 项目需要强化安全管理，特别是对 Skills 文件中的敏感凭证管理
- 存在潜在的安全风险，如硬编码的 API Key、明文 OAuth 凭证等
- 需要建立自动化的安全检查机制

### 2.2 实施过程
1. **安全评估**：运行安全验证脚本，识别潜在安全问题
2. **问题修复**：修复安全验证脚本中的误报问题
3. **流程集成**：将安全检查集成到 Git 工作流和 CI/CD 流水线
4. **标准制定**：建立安全标准和规范
5. **团队通知**：向团队成员传达新的安全要求

### 2.3 遇到的挑战与解决方案

| 挑战 | 解决方案 | 经验教训 |
|------|---------|---------|
| 安全验证脚本误报 | 添加智能排除规则，排除示例代码中的模式 | 安全工具需要考虑示例代码的特殊性 |
| Git hook 集成 | 在现有 hook 基础上添加安全检查 | 保持现有功能的同时，添加新功能 |
| 团队意识不足 | 创建详细的安全文档和培训材料 | 定期培训和提醒是保持安全意识的关键 |

## 3. 安全最佳实践

### 3.1 凭证管理
- **使用占位符**：所有敏感凭证必须使用 `[UPPER_CASE_NAME]` 格式
- **环境变量**：通过 `process.env.XXX` 读取配置
- **模板文件**：`.env.skills.template` 仅包含占位符和说明
- **Git 忽略**：确保敏感文件（如 `.env.local`）被添加到 `.gitignore`

### 3.2 代码安全
- **输入验证**：对所有用户输入进行验证和 sanitize
- **密码存储**：使用强哈希算法存储密码，避免明文存储
- **API 调用**：使用 HTTPS，避免在 URL 中传递敏感信息
- **错误处理**：避免在错误消息中暴露敏感信息
- **日志安全**：确保日志中不记录完整的敏感信息

### 3.3 依赖管理
- **定期更新**：定期更新依赖包，修复已知漏洞
- **漏洞扫描**：使用 `npm audit` 等工具扫描依赖漏洞
- **最小权限**：只安装必要的依赖包

### 3.4 部署安全
- **环境隔离**：开发、测试、生产环境分离
- **访问控制**：严格的权限管理和访问控制
- **配置管理**：使用环境变量或配置管理系统管理敏感配置

## 4. 常见安全漏洞类型

### 4.1 注入攻击
- **SQL 注入**：通过用户输入操纵 SQL 查询
- **NoSQL 注入**：类似 SQL 注入，但针对 NoSQL 数据库
- **命令注入**：通过用户输入执行恶意命令

### 4.2 认证与授权
- **弱密码**：使用容易猜测的密码
- **会话管理**：不安全的会话处理
- **权限提升**：未正确限制用户权限

### 4.3 敏感数据暴露
- **明文存储**：明文存储密码、API Key 等敏感信息
- **不安全传输**：使用 HTTP 而非 HTTPS
- **日志泄露**：在日志中记录敏感信息

### 4.4 安全配置错误
- **默认配置**：使用默认的安全配置
- **过度权限**：给予应用程序过多权限
- **错误信息**：暴露详细的错误信息

## 5. 安全工具使用指南

### 5.1 validate-security.sh
- **功能**：快速验证 Skills 文件中是否包含明文敏感凭证
- **使用方法**：
  ```bash
  # 验证 skills 目录
  bash skills/security/validate-security.sh
  
  # 验证指定目录
  bash skills/security/validate-security.sh ./src
  
  # 详细模式
  bash skills/security/validate-security.sh --verbose
  ```
- **退出码**：
  - 0：验证通过
  - 1：有警告
  - 2：验证失败

### 5.2 GitHub Actions 安全扫描
- **触发时机**：
  - 代码推送到 main 或 develop 分支
  - 拉取请求
  - 每月 1 日自动运行
- **检查内容**：
  - 安全验证
  - 依赖扫描
  - gitignore 规则检查
- **报告**：自动生成并上传安全报告

### 5.3 其他安全工具
- **npm audit**：扫描依赖包漏洞
- **GitLeaks**：检测代码中的敏感信息
- **SonarQube**：代码质量和安全分析

## 6. 案例分析

### 6.1 案例一：硬编码 API Key
- **问题**：在代码中硬编码 OpenAI API Key
- **风险**：API Key 泄露，导致未授权使用和费用损失
- **解决方案**：使用环境变量存储 API Key，在代码中通过 `process.env` 读取
- **预防措施**：使用安全验证工具定期检查，集成到 CI/CD 流程

### 6.2 案例二：明文 OAuth 凭证
- **问题**：在 SKILL.md 文件中明文存储 OAuth Client ID 和 Client Secret
- **风险**：凭证泄露，导致第三方服务被未授权访问
- **解决方案**：使用 `[CLIENT_ID]` 和 `[CLIENT_SECRET]` 占位符，通过环境变量注入
- **预防措施**：建立凭证管理规范，定期审计

### 6.3 案例三：不安全的数据库连接串
- **问题**：在配置文件中包含真实的数据库密码
- **风险**：数据库被未授权访问，数据泄露
- **解决方案**：使用 `[DB_PASSWORD]` 占位符，通过环境变量注入
- **预防措施**：使用安全验证工具检查配置文件
