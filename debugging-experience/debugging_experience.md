# 调试经验与最佳实践

## 技术领域：SSH配置

### 经验1：端口冲突排查
- **问题场景**：服务无法启动，提示端口被占用
- **核心思路**：使用 `netstat -tulpn` 检查端口占用情况
- **解决方案**：
  - 识别占用端口的进程
  - 检查配置文件中是否有重复的端口配置
  - 移除不必要的端口配置
- **最佳实践**：
  - SSH配置中只保留必要的端口（通常是22）
  - 定期检查端口占用情况
  - 使用非标准端口时确保与其他服务不冲突

### 经验2：配置变更管理
- **问题场景**：配置修改后服务异常
- **核心思路**：修改配置前先备份，修改后验证配置语法
- **解决方案**：
  - `sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak`
  - 修改后使用 `sudo sshd -t` 验证配置
  - 重启服务并检查状态
- **最佳实践**：
  - 所有配置修改都应有备份
  - 修改前评估影响范围
  - 建立配置变更记录

## 技术领域：Nginx配置

### 经验3：权限问题诊断
- **问题场景**：Nginx返回500错误，日志显示权限拒绝
- **核心思路**：检查文件和目录权限，确保Nginx进程用户可访问
- **解决方案**：
  - 检查目录权限：`ls -la /path/to/directory`
  - 检查文件权限：`stat -c "%a %U:%G" /path/to/file`
  - 测试访问：`sudo -u www-data stat /path/to/file`
- **最佳实践**：
  - 目录权限至少为755
  - 文件权限至少为644
  - Nginx进程用户应有访问权限

### 经验4：HTTPS配置优化
- **问题场景**：HTTPS服务无法启动或性能不佳
- **核心思路**：确保SSL证书正确配置，优化SSL参数
- **解决方案**：
  - 验证证书文件存在且权限正确
  - 使用现代化的SSL协议和密码套件
  - 启用HTTP/2和HSTS
- **最佳实践**：
  - 使用Let's Encrypt免费证书
  - 定期更新证书
  - 配置适当的SSL安全参数

### 经验5：配置语法验证
- **问题场景**：Nginx配置错误导致服务无法启动
- **核心思路**：修改配置后及时验证语法
- **解决方案**：
  - `sudo nginx -t` 验证配置语法
  - 检查错误信息并修复
  - 重载配置：`sudo systemctl reload nginx`
- **最佳实践**：
  - 任何配置修改后都应验证
  - 分阶段测试配置变更
  - 保持配置文件整洁有序

## 技术领域：文件系统权限

### 经验6：权限继承问题
- **问题场景**：文件存在但无法访问
- **核心思路**：权限问题不仅涉及文件本身，还涉及整个路径
- **解决方案**：
  - 检查完整路径的权限：`namei -l /path/to/file`
  - 确保每个目录都有执行权限（x）
  - 递归修复权限：`chmod -R 755 /path/to/directory`
- **最佳实践**：
  - 从顶层目录开始检查权限
  - 使用最小权限原则
  - 统一文件权限管理策略

### 经验7：用户和组管理
- **问题场景**：服务进程无法访问文件
- **核心思路**：确保服务进程用户对文件有适当权限
- **解决方案**：
  - 查看服务进程用户：`ps aux | grep service_name`
  - 更改文件所有者：`chown -R user:group /path/to/files`
  - 检查并调整权限
- **最佳实践**：
  - 为服务创建专用用户
  - 避免使用root用户运行服务
  - 建立统一的用户权限管理流程

## 技术领域：调试方法论

### 经验8：系统性排查流程
- **问题场景**：复杂的服务故障
- **核心思路**：按步骤系统性排查，从症状到根因
- **解决方案**：
  1. 检查服务状态：`systemctl status service`
  2. 查看错误日志：`journalctl -u service` 或应用日志
  3. 检查网络连接：`netstat`, `ping`, `curl`
  4. 验证配置：语法检查，配置文件验证
  5. 测试基础功能：最小化测试用例
- **最佳实践**：
  - 建立标准化的排查流程
  - 记录排查过程和结果
  - 从最简单的测试开始

### 经验9：日志分析技巧
- **问题场景**：服务异常但原因不明
- **核心思路**：通过日志定位问题根源
- **解决方案**：
  - 查看最新日志：`tail -n 50 /var/log/service.log`
  - 搜索错误关键词：`grep "error" /var/log/service.log`
  - 分析时间序列：按时间顺序关联事件
  - 对比正常和异常状态的日志
- **最佳实践**：
  - 配置适当的日志级别
  - 建立日志集中管理
  - 定期清理和归档日志

### 经验10：测试验证策略
- **问题场景**：修复后需要验证解决方案
- **核心思路**：从不同角度验证修复效果
- **解决方案**：
  - 本地测试：`curl http://localhost/service`
  - 远程测试：`curl https://domain.com`
  - 功能测试：验证核心功能
  - 性能测试：检查响应时间
- **最佳实践**：
  - 建立自动化测试脚本
  - 测试覆盖各种场景
  - 记录测试结果和基准数据

## 通用最佳实践

### 1. 备份与恢复
- **实践**：任何配置修改前都应备份
- **方法**：`cp original.conf original.conf.bak`
- **理由**：快速回滚错误配置，减少故障时间

### 2. 变更管理
- **实践**：记录所有配置变更和操作
- **方法**：使用版本控制或变更日志
- **理由**：追踪变更历史，便于问题定位

### 3. 最小化变更
- **实践**：每次只进行必要的最小变更
- **方法**：分步骤实施，每次验证一个变更
- **理由**：快速定位问题根源，减少影响范围

### 4. 文档化
- **实践**：记录问题、解决方案和经验
- **方法**：建立知识库和故障案例库
- **理由**：知识共享，避免重复解决相同问题

### 5. 预防性维护
- **实践**：定期检查系统状态和配置
- **方法**：建立巡检制度和监控系统
- **理由**：提前发现潜在问题，防患于未然
