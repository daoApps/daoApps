## 快速检索指南

### 问题：服务无法启动
1. 检查服务状态：`systemctl status service`
2. 查看错误日志：`journalctl -u service`
3. 检查端口占用：`netstat -tulpn`
4. 验证配置：服务特定的配置检查命令

### 问题：权限错误
1. 检查文件权限：`ls -la file`
2. 检查目录权限：`ls -la directory`
3. 测试访问：`sudo -u user stat file`
4. 修复权限：`chmod` 和 `chown` 命令

### 问题：网络连接
1. 测试本地连接：`curl http://localhost`
2. 测试远程连接：`curl https://domain`
3. 检查网络状态：`ping`、`traceroute`
4. 检查防火墙：`iptables -L`

### 问题：配置错误
1. 验证配置语法：服务特定的验证命令
2. 检查配置文件：查看关键配置项
3. 恢复备份：如果修改导致问题
4. 逐步测试：分步骤验证配置变更
