# 调试经验知识体系

## 项目概述

本知识体系记录了在部署任务中积累的调试经验和最佳实践，旨在帮助团队成员快速解决类似问题，提高调试效率。

## 文档结构

### 1. 调试过程数据
- **文件**: `debugging_data.md`
- **内容**: 详细的调试过程记录，包括时间线、问题描述、解决方案和关键命令
- **用途**: 提供完整的历史记录，便于理解问题的发生和解决过程

### 2. 调试经验与最佳实践
- **文件**: `debugging_experience.md`
- **内容**: 按技术领域分类的调试经验和最佳实践
- **用途**: 提供可复用的解决思路和方法

### 3. 问题-解决方案映射
- **文件**: `problem_solution_mapping.md`
- **内容**: 结构化的问题类型与解决方案映射表
- **用途**: 快速检索和定位解决方案

## 技术领域覆盖

- **SSH配置**: 端口管理、配置验证、安全设置
- **Nginx配置**: 权限管理、HTTPS配置、性能优化
- **文件系统权限**: 权限管理、用户和组配置
- **网络连接**: 连接测试、故障排查
- **服务管理**: 启动故障、性能监控

## 快速使用指南

### 遇到问题时
1. **识别问题类型**: 根据错误症状和技术领域
2. **查找解决方案**: 参考问题-解决方案映射表
3. **应用经验**: 使用调试经验文档中的方法
4. **验证结果**: 按照测试验证策略确认修复效果

### 贡献新经验
1. **记录问题**: 详细描述问题症状和环境
2. **记录解决方案**: 包括具体步骤和命令
3. **提取经验**: 总结可复用的解决思路
4. **更新映射**: 将新问题和解决方案添加到映射表

## 附录

### 常用调试命令

#### 服务状态检查
```bash
systemctl status [service]
systemctl restart [service]
```

#### 日志查看
```bash
tail -n 50 /var/log/[service].log
journalctl -u [service]
```

#### 网络检查
```bash
netstat -tulpn
ping [host]
curl [url]
```

#### 权限检查
```bash
ls -la [path]
stat -c "%a %U:%G" [file]
sudo -u [user] stat [file]
```

#### 配置验证
```bash
nginx -t
sshd -t
```
