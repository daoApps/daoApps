   - 使用了 `parserOptions.project: './tsconfig.app.json'`
   - 这要求所有被 lint 的 TypeScript/Vue 文件必须在 tsconfig.app.json 的 `include` 范围内

**采用方案 A（推荐）**：在 ESLint 配置中排除 `packages/` 目录

**理由**：

## 🔬 3. 根因分析

### 问题本质
**TypeScript 项目配置与 ESLint 配置范围不匹配**

### 详细原因链

1️⃣ **ESLint 配置使用了严格的项目模式**
   - [eslint.config.js#L25](file:///d:/xinet/daoCollective/daoApps/daoApps/apps/AgentPit/eslint.config.js#L25) 配置了：
     ```javascript
     parserOptions: {
       project: ['./tsconfig.app.json']  // 指定使用 tsconfig.app.json 进行类型检查
     }
     ```

2️⃣ **tsconfig.app.json 的包含范围有限**
   - [tsconfig.app.json](file:///d:/xinet/daoCollective/daoApps/daoApps/apps/AgentPit/tsconfig.app.json) 只配置了：
     ```json
     "include": ["src/**/*.ts", "src/**/*.tsx", "src/**/*.vue"]
     ```
   - **未包含**: `packages/ui/**` 目录

3️⃣ **monorepo 结构导致的冲突**
   - 项目采用 monorepo 结构，存在独立的 `packages/ui` 子包
   - ESLint 默认扫描整个项目目录（包括 `packages/`）
   - 当扫描到 `packages/ui/` 下的 TypeScript/Vue 文件时，尝试使用 `tsconfig.app.json` 进行类型检查
   - 由于这些文件不在 tsconfig 的 `include` 范围内，导致解析失败

### 1. Monorepo 项目管理建议
如果未来需要对 `packages/ui` 进行代码质量检查，建议：

**选项 A**: 为 `packages/ui` 创建独立的 ESLint 配置
```bash
# 在 packages/ui 目录下创建
packages/ui/eslint.config.js
```

**选项 B**: 创建项目级别的 tsconfig 包含所有目录
```json
// tsconfig.json (根级别)
{
  "references": [
    { "path": "./tsconfig.app.json" },
    { "path": "./packages/ui/tsconfig.json" }
  ]
}
```

### 2. 配置维护建议
- 定期检查 `ignores` 列表，确保与新添加的目录保持同步
- 在 `.gitignore` 和 ESLint `ignores` 之间保持一致性
- 考虑在项目 README 中记录 monorepo 的 lint 策略

### 3. CI/CD 集成建议
如需在 CI 中同时检查主项目和子包，可配置多阶段 lint：
```yaml
# 示例：GitHub Actions
- name: Lint Main Project
  run: npm run lint:check
  
- name: Lint UI Package
  working-directory: packages/ui
  run: npm run lint:check  # 如果配置了独立 lint
```
